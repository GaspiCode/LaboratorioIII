Seccion de Preguntas sobre HTTP

  • 1. ¿Sobre que protocolo binario esta montado el protocolo HTTP

    El protocolo HTTP esta montado sobre el protocolo TCP, este establecera conexiones entre el cliente y el servidor, y garantizara la entrega confiable de datos a través de la red. HTTP estaria, relativamente al modelo OSI, en la capa de aplicacion, mientras que TCP estaria en la capa de transporte.

  • 2. ¿Cuales son los clientes HTTP y los servidores HTTP mas utilizados?

    Los clientes HTTP mas utilizados son Postman, cURL e Insomnia, mientras que los servidores HTTP mas usados son Apache, Nginx y Microsoft Internet Information Services (IIS).

  • 3. ¿Que verbos admite un comando en el requerimiento HTTP?

    Los verbos mas utilizados son GET, POST, PUT, DELETE y PATCH, pero tambien existen otros mas especializados como HEAD, OPTIONS, TRACE y CONNECT.

  • 4. ¿Que contenido lleva el body de un requerimiento HTTP?

    Para la transmision de datos desde el cliente al server, se envia un request. Este request puede ser un GET o POST, PUT, PATCH, etc. En el caso de GET los datos viajan como variables declaradas en la url. En el caso de POST u otros los datos viajan en el body. El request HTTP puede llevar en el body datos, ya sean Binarios, Texto, Streams(Audio, Video, etc), Archivos, JSON, XML, entre otros. Pueden utilizarse, por ejemplo, para hacer operaciones CRUD en una base de datos.

  • 5. ¿Que diferencia existe entre un URL y un URI?

    La diferencia entre un URL y un URI es que el URI es un identificador unico para un recurso web relativo a una conexion TCP ya establecida; mientras que la URL es un identificador unico para un sitio web, con este identificador se establecera la conexion TCP ya mencionada.

  • 6. ¿Como se almacena información relacionada con las respuestas http?

    Depende de las caracteristicas del proyecto, por ejemplo, si mi respuesta es un flujo de datos por que estoy mirando Youtube la informacion se ira almacenando en una cache de cierta manera; en cambio, si mi respuesta es una booleana que me indica si se inserto un registro en una db, puede llegar a guardarse en un logfile. Otro caso podria ser el inverso, hice un request a un REST y este me devuelve como respuesta un JSON denotando los datos de mi consulta.

  • 7. ¿Que significa Virtual hosting?

    Virtual Hosting se refiere a la capacidad de un servidor web de alojar varios sitios web en el mismo server fisico. HTTP 1.1 tiene la capacidad de correr multiples dominios en una sola direccion IP.

  • 8. ¿Porque el HTTP se considera STATELESS?

    El protocolo HTTP se considera Stateless ("Sin estado") por que no tiene la capacidad por si mismo de perseverar estados al cerrar la conexion, una vez que la conexion se pierde HTTP no mantiene ninguna información sobre el estado de las comunicaciones entre el cliente y el servidor. Cada solicitud HTTP que realiza un cliente es independiente y no tiene conocimiento de las solicitudes anteriores o futuras realizadas por el mismo cliente.

  • 9. ¿Como va a mejorar la velocidad de la WEB en poco tiempo?

    La velocidad de la web puede llegar a mejorar gracias a la implementacion de una mejora sobre HTTP 1.1, HTTP 2.0. Esta actualizacion trae consigo requerimientos multiplexados (puede hacer varias request sin tener que esperar respuestas para hacerlas), entregas multiplexadas (se pueden hacer varias entregas de datos al mismo tiempo) y transmicion de datos en formato de paquetes binarios.

  • 10. ¿Como va a mejorar el comportamiento de una aplicación WEB en ambientes ruidosos o de débil conexión?

    Se puede mejorar el comportamiento de un sitio web en ambientes ruidosos con una nueva implementacion de HTTP, HTTP 3.0. Esta nueva version esta montada sobre un protocol QUIC y un protocolo UDP, y ofrece una gran velocidad asi como una gran capacidad de reconexion, mejorando la conexion en ambientes ruidosos, como por ejemplo, un aeropuerto.
